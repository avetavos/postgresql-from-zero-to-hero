array มีอยู่ทุกที่ในเอกสาร MongoDB — แท็กบนโพสต์ รายการในออเดอร์ บทเรียนในคอร์ส การจะ *สรุป* ข้ามสมาชิกของ array เหล่านั้น มักต้องดูทีละตัว `$unwind` ทำสิ่งนั้นพอดี: มันรับเอกสารที่มีฟิลด์ array แล้วปล่อยออกมา **หนึ่งเอกสารต่อหนึ่งสมาชิก** โดยคัดลอกส่วนที่เหลือของเอกสารไปยังแต่ละอัน เอกสารที่มี array สามสมาชิกจะกลายเป็นสามเอกสาร

สำหรับบทเรียนนี้ เอกสาร `sales` แต่ละอันบันทึกออเดอร์ที่มีหลายรายการสินค้าใน array `items`:

```json
{
  "_id": "o1",
  "buyer": "Ada",
  "items": [
    { "title": "Dune", "qty": 2 },
    { "title": "Foundation", "qty": 1 }
  ]
}
```

## หนึ่งเอกสารต่อหนึ่งสมาชิก array

ให้เส้นทางไปยังฟิลด์ array แก่ `$unwind` (พร้อมตัวนำหน้า `$`) แล้วมันจะแผ่เอกสารออก เอกสารที่ปล่อยออกมาทุกอันเหมือนกันทุกประการ ยกเว้นฟิลด์ array ถูกแทนที่ด้วยสมาชิกตัวใดตัวหนึ่งของมัน

ออเดอร์เดียวกลายเป็นหนึ่งเอกสารต่อหนึ่งรายการสินค้า โดย `items` ตอนนี้เป็นออบเจกต์ธรรมดาแทนที่จะเป็น array:

```json
[
  { "_id": "o1", "buyer": "Ada", "items": { "title": "Dune", "qty": 2 } },
  { "_id": "o1", "buyer": "Ada", "items": { "title": "Foundation", "qty": 1 } }
]
```

**ใน Compass:** เพิ่ม stage `$unwind` ในแท็บ **Aggregations** แล้วชี้มันไปที่ฟิลด์ array การดูตัวอย่างแสดงจำนวนเอกสาร *เพิ่มขึ้น* — สัญญาณว่าคุณได้แบน array แล้ว — ซึ่งตรงข้ามกับสิ่งที่ `$match` และ `$group` ทำ

## การจับคู่คลาสสิก: $unwind แล้วตามด้วย $group

`$unwind` ไม่ค่อยเป็นคำพูดสุดท้าย งานปกติของมันคือแบน array เพื่อให้ `$group` ในภายหลังสรุปข้ามสมาชิกได้ ที่นี่เราแบน items ของทุกออเดอร์ แล้วรวมจำนวนที่ขายต่อชื่อเรื่องข้ามทุกออเดอร์:

<Tabs syncKey="lang">
  <TabItem label="mongosh">
    ```js
    db.sales.aggregate([
      { $unwind: "$items" },
      { $group: {
          _id: "$items.title",
          sold: { $sum: "$items.qty" }
      } },
      { $sort: { sold: -1 } }
    ])
    ```
  </TabItem>
  <TabItem label="Node.js">
    ```js
    const rows = await db.collection("sales").aggregate([
      { $unwind: "$items" },
      { $group: {
          _id: "$items.title",
          sold: { $sum: "$items.qty" },
      } },
      { $sort: { sold: -1 } },
    ]).toArray();
    ```
  </TabItem>
  <TabItem label="Python">
    ```python
    rows = list(db.sales.aggregate([
        {"$unwind": "$items"},
        {"$group": {
            "_id": "$items.title",
            "sold": {"$sum": "$items.qty"},
        }},
        {"$sort": {"sold": -1}},
    ]))
    ```
  </TabItem>
  <TabItem label="Go">
    ```go
    pipeline := mongo.Pipeline{
        {{"$unwind", "$items"}},
        {{"$group", bson.D{
            {"_id", "$items.title"},
            {"sold", bson.D{{"$sum", "$items.qty"}}},
        }}},
        {{"$sort", bson.D{{"sold", -1}}}},
    }
    cursor, err := coll.Aggregate(ctx, pipeline)
    if err != nil {
        return err
    }
    ```
  </TabItem>
  <TabItem label="Rust">
    ```rust
    let pipeline = vec![
        doc! { "$unwind": "$items" },
        doc! { "$group": {
            "_id": "$items.title",
            "sold": { "$sum": "$items.qty" }
        } },
        doc! { "$sort": { "sold": -1 } },
    ];
    let mut cursor = coll.aggregate(pipeline).await?;
    ```
  </TabItem>
</Tabs>

ตอนนี้แต่ละชื่อเรื่องมีจำนวนรวมของมัน รวมข้ามรายการสินค้าของทุกออเดอร์:

```json
[
  { "_id": "Dune", "sold": 5 },
  { "_id": "Foundation", "sold": 3 }
]
```

## การเก็บ array ที่ว่างและที่หายไป

โดยค่าเริ่มต้น `$unwind` **ทิ้ง** เอกสารที่ฟิลด์ array ว่าง หายไป หรือเป็น `null` — ไม่มีสมาชิกให้ปล่อยออก ดังนั้นเอกสารจึงหายไป เมื่อคุณต้องการให้เอกสารเหล่านั้นรอด (โดยไม่มีฟิลด์นั้น) ให้ส่งรูปแบบยาวพร้อม `preserveNullAndEmptyArrays`:

<Tabs syncKey="lang">
  <TabItem label="mongosh">
    ```js
    db.sales.aggregate([
      { $unwind: {
          path: "$items",
          preserveNullAndEmptyArrays: true
      } }
    ])
    ```
  </TabItem>
  <TabItem label="Node.js">
    ```js
    const rows = await db.collection("sales").aggregate([
      { $unwind: {
          path: "$items",
          preserveNullAndEmptyArrays: true,
      } },
    ]).toArray();
    ```
  </TabItem>
  <TabItem label="Python">
    ```python
    rows = list(db.sales.aggregate([
        {"$unwind": {
            "path": "$items",
            "preserveNullAndEmptyArrays": True,
        }},
    ]))
    ```
  </TabItem>
  <TabItem label="Go">
    ```go
    pipeline := mongo.Pipeline{
        {{"$unwind", bson.D{
            {"path", "$items"},
            {"preserveNullAndEmptyArrays", true},
        }}},
    }
    cursor, err := coll.Aggregate(ctx, pipeline)
    if err != nil {
        return err
    }
    ```
  </TabItem>
  <TabItem label="Rust">
    ```rust
    let pipeline = vec![
        doc! { "$unwind": {
            "path": "$items",
            "preserveNullAndEmptyArrays": true
        } },
    ];
    let mut cursor = coll.aggregate(pipeline).await?;
    ```
  </TabItem>
</Tabs>

ออเดอร์ที่ไม่มี items ยังคงปรากฏหนึ่งครั้ง เพียงแต่ไม่มีฟิลด์ `items`:

```json
[
  { "_id": "o2", "buyer": "Grace" }
]
```

## ทิปและกับดัก

- `$unwind` **ทวีคูณ** เอกสาร ออเดอร์ที่มีห้ารายการกลายเป็นห้าเอกสาร — การแบนสามารถทำให้สตรีมโตขึ้นมากก่อนที่ `$group` ในภายหลังจะทำให้มันหดลงอีกครั้ง
- หากไม่มี `preserveNullAndEmptyArrays` เอกสารที่มี array ว่างหรือหายไปจะถูกทิ้งเงียบ ๆ ถ้าจำนวนนับของคุณขาดไปหลัง unwind นี่คือผู้ต้องสงสัยตามปกติ
- หลังจาก unwind ให้อ้างถึงฟิลด์ของสมาชิกผ่านเส้นทางเดิม: เมื่อ `items` เป็นออบเจกต์เดี่ยว `"$items.qty"` จะอ่านจำนวนของสมาชิกนั้น
- ถ้าฟิลด์ไม่ได้เป็น array จริง ๆ `$unwind` จะปฏิบัติต่อค่าสเกลาร์เดี่ยวเสมือนเป็น array หนึ่งสมาชิก และปล่อยเอกสารออกมาหนึ่งครั้ง — ดังนั้นมันให้อภัยกับค่าที่ไม่ใช่ array

<Callout title="unwind แผ่ออก แล้ว group พับกลับ">
$unwind เปลี่ยนหนึ่งเอกสารที่มี array N สมาชิกให้เป็น N เอกสาร โดยลำพังมันแค่ทำให้สตรีมพองตัว พลังที่แท้จริงของมันปรากฏเมื่อ $group ตามมาเพื่อสรุปข้ามสมาชิกที่ถูกแบนทั้งหมดเหล่านั้น ตั้ง preserveNullAndEmptyArrays เมื่อเอกสารที่มี array ว่างต้องรอด
</Callout>
