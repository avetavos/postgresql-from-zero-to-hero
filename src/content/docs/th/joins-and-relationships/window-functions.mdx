`GROUP BY` ยุบหลายแถวให้กลายเป็นแถวสรุปแถวเดียว และบางครั้งนั่นก็ผิดเป๊ะ ๆ: คุณต้องการทั้งบทสรุป *และ* รายละเอียดต้นฉบับวางเคียงข้างกัน window function ทำเช่นนั้น window function มองไปยังชุดของแถวที่เกี่ยวข้องกับแถวปัจจุบัน — *window* ของมัน — แล้วคำนวณค่าออกมา แต่มันคงทุกแถวไว้ในผลลัพธ์ นั่นคือวิธีที่คุณเพิ่มอันดับ, ยอดสะสม, หรือค่าเฉลี่ยต่อกลุ่มเป็นคอลัมน์เพิ่มเติมโดยไม่สูญเสียรายละเอียดใด ๆ

เรายังคงใช้ตาราง `authors`, `books`, และ `sales` ต่อไป

## OVER clause

คุณเปลี่ยนฟังก์ชันธรรมดาให้กลายเป็น window function โดยการเพิ่ม `OVER` clause ภายในนั้น `PARTITION BY` แบ่งแถวออกเป็นกลุ่ม (เหมือน `GROUP BY` แต่ไม่ยุบมัน) และ `ORDER BY` กำหนดลำดับภายในแต่ละ partition ที่นี่เราใส่หมายเลขให้ยอดขายภายในแต่ละภูมิภาค โดยเรียงจากมากสุดก่อน

<Tabs syncKey="lang">
  <TabItem label="psql">
    ```sql
    SELECT region, copies,
           ROW_NUMBER() OVER (PARTITION BY region ORDER BY copies DESC) AS seq
    FROM sales
    ORDER BY region, seq;
    ```
  </TabItem>
  <TabItem label="Node.js">
    ```js
    const res = await pool.query(`
      SELECT region, copies,
             ROW_NUMBER() OVER (PARTITION BY region ORDER BY copies DESC) AS seq
      FROM sales
      ORDER BY region, seq
    `);
    ```
  </TabItem>
  <TabItem label="Python">
    ```python
    cur.execute("""
        SELECT region, copies,
               ROW_NUMBER() OVER (PARTITION BY region ORDER BY copies DESC) AS seq
        FROM sales
        ORDER BY region, seq
    """)
    rows = cur.fetchall()
    ```
  </TabItem>
  <TabItem label="Go">
    ```go
    rows, err := conn.Query(ctx, `
      SELECT region, copies,
             ROW_NUMBER() OVER (PARTITION BY region ORDER BY copies DESC) AS seq
      FROM sales
      ORDER BY region, seq`)
    ```
  </TabItem>
  <TabItem label="Rust">
    ```rust
    let rows = sqlx::query(
        "SELECT region, copies,
                ROW_NUMBER() OVER (PARTITION BY region ORDER BY copies DESC) AS seq
         FROM sales
         ORDER BY region, seq",
    )
    .fetch_all(&pool)
    .await?;
    ```
  </TabItem>
</Tabs>

```text
 region | copies | seq
--------+--------+-----
 East   |    900 |   1
 East   |    200 |   2
 West   |    400 |   1
 West   |    350 |   2
```

การใส่หมายเลขเริ่มต้นใหม่ที่ 1 ภายในแต่ละภูมิภาคเพราะ `PARTITION BY region` ทุกแถวของยอดขายยังคงอยู่ครบ — ไม่มีอะไรถูกยุบ ตรงกันข้ามกับ `GROUP BY` ที่จะเหลือเพียงแถวเดียวต่อหนึ่งภูมิภาค

## ROW_NUMBER, RANK, และ DENSE_RANK

ฟังก์ชันจัดอันดับสามตัวนี้ล้วนเรียงลำดับแถวภายใน partition แต่ต่างกันที่วิธีจัดการกับค่าที่เสมอกัน `ROW_NUMBER` ให้หมายเลขที่ไม่ซ้ำกันเสมอ `RANK` ให้แถวที่เสมอกันได้หมายเลขเดียวกันแล้วกระโดดข้ามไปข้างหน้า `DENSE_RANK` ให้แถวที่เสมอกันได้หมายเลขเดียวกันแต่ไม่กระโดดข้าม

<Tabs syncKey="lang">
  <TabItem label="psql">
    ```sql
    SELECT copies,
           ROW_NUMBER()  OVER (ORDER BY copies DESC) AS row_num,
           RANK()        OVER (ORDER BY copies DESC) AS rnk,
           DENSE_RANK()  OVER (ORDER BY copies DESC) AS dense
    FROM sales
    ORDER BY copies DESC;
    ```
  </TabItem>
  <TabItem label="Node.js">
    ```js
    const res = await pool.query(`
      SELECT copies,
             ROW_NUMBER() OVER (ORDER BY copies DESC) AS row_num,
             RANK()       OVER (ORDER BY copies DESC) AS rnk,
             DENSE_RANK() OVER (ORDER BY copies DESC) AS dense
      FROM sales
      ORDER BY copies DESC
    `);
    ```
  </TabItem>
  <TabItem label="Python">
    ```python
    cur.execute("""
        SELECT copies,
               ROW_NUMBER() OVER (ORDER BY copies DESC) AS row_num,
               RANK()       OVER (ORDER BY copies DESC) AS rnk,
               DENSE_RANK() OVER (ORDER BY copies DESC) AS dense
        FROM sales
        ORDER BY copies DESC
    """)
    rows = cur.fetchall()
    ```
  </TabItem>
  <TabItem label="Go">
    ```go
    rows, err := conn.Query(ctx, `
      SELECT copies,
             ROW_NUMBER() OVER (ORDER BY copies DESC) AS row_num,
             RANK()       OVER (ORDER BY copies DESC) AS rnk,
             DENSE_RANK() OVER (ORDER BY copies DESC) AS dense
      FROM sales
      ORDER BY copies DESC`)
    ```
  </TabItem>
  <TabItem label="Rust">
    ```rust
    let rows = sqlx::query(
        "SELECT copies,
                ROW_NUMBER() OVER (ORDER BY copies DESC) AS row_num,
                RANK()       OVER (ORDER BY copies DESC) AS rnk,
                DENSE_RANK() OVER (ORDER BY copies DESC) AS dense
         FROM sales
         ORDER BY copies DESC",
    )
    .fetch_all(&pool)
    .await?;
    ```
  </TabItem>
</Tabs>

```text
 copies | row_num | rnk | dense
--------+---------+-----+-------
    900 |       1 |   1 |     1
    400 |       2 |   2 |     2
    400 |       3 |   2 |     2
    200 |       4 |   4 |     3
    150 |       5 |   5 |     4
    50  |       6 |   6 |     5
```

ดูสองแถวของ 400 สิ `ROW_NUMBER` เรียกมันว่า 2 และ 3, `RANK` เรียกทั้งคู่ว่า 2 แล้วกระโดดไป 4, ส่วน `DENSE_RANK` เรียกทั้งคู่ว่า 2 แล้วไปต่อที่ 3 เลือกตัวที่พฤติกรรมต่อค่าเสมอกันตรงกับที่คุณต้องการ

## ยอดสะสมด้วย SUM() OVER

เพิ่ม `ORDER BY` ภายใน `OVER` ให้กับ aggregate แล้วมันจะกลายเป็นการคำนวณแบบสะสม: แต่ละแถวจะเห็นทั้งตัวมันเองบวกกับทุกอย่างที่อยู่ก่อนหน้ามันตามลำดับ นี่คือวิธีมาตรฐานในการผลิตยอดสะสม

<Tabs syncKey="lang">
  <TabItem label="psql">
    ```sql
    SELECT region, copies,
           SUM(copies) OVER (PARTITION BY region ORDER BY copies DESC) AS running
    FROM sales
    ORDER BY region, copies DESC;
    ```
  </TabItem>
  <TabItem label="Node.js">
    ```js

      SELECT region, copies,
             SUM(copies) OVER (PARTITION BY region ORDER BY copies DESC) AS running
      FROM sales

    `);
    ```
  </TabItem>
  <TabItem label="Python">
    ```python
    cur.execute("""
        SELECT region, copies,
               SUM(copies) OVER (PARTITION BY region ORDER BY copies DESC) AS running
        FROM sales
        ORDER BY region, copies DESC
    """)
    rows = cur.fetchall()
    ```
  </TabItem>
  <TabItem label="Go">
    ```go
    rows, err := conn.Query(ctx, `
      SELECT region, copies,
             SUM(copies) OVER (PARTITION BY region ORDER BY copies DESC) AS running
      FROM sales
      ORDER BY region, copies DESC`)
    ```
  </TabItem>
  <TabItem label="Rust">
    ```rust

        "SELECT region, copies,
                SUM(copies) OVER (PARTITION BY region ORDER BY copies DESC) AS running
         FROM sales
         ORDER BY region, copies DESC",
    )
    .fetch_all(&pool)
    .await?;
    ```
  </TabItem>
</Tabs>

```text
 region | copies | running
--------+--------+---------
 East   |    900 |     900
 East   |    200 |    1100
 West   |    400 |     400
 West   |    350 |     750
```

คอลัมน์ `running` สะสมลงไปในแต่ละภูมิภาคและรีเซ็ตที่ขอบเขตของ partition หากไม่มี `ORDER BY` ภายใน `OVER` คำสั่ง `SUM(copies) OVER (PARTITION BY region)` จะกลับไปแสดงยอดรวมทั้งหมดของภูมิภาคซ้ำในทุกแถวแทน — เทคนิคที่มีประโยชน์สำหรับการแสดงยอดขายแต่ละรายการเคียงข้างยอดรวมของภูมิภาค

## window เก็บแถว ส่วน GROUP BY ยุบมัน

ภาพด้านล่างเปรียบเทียบทั้งสองอย่าง `GROUP BY region` จะคืนสองแถว หนึ่งแถวต่อหนึ่งภูมิภาค ส่วน window function แบ่ง partition จากแถวชุดเดียวกันแต่คืนมันทั้งหมด แต่ละแถวพกค่าต่อ partition ของมันมาด้วย

## ทิป / จุดพลาดที่พบบ่อย

- window function ไม่เคยลดจำนวนแถว ถ้าคุณต้องการหนึ่งแถวต่อหนึ่งกลุ่ม ให้ใช้ `GROUP BY` ถ้าคุณต้องการทุกแถวพร้อมตัวเลขต่อกลุ่ม ให้ใช้ window
- `PARTITION BY` ใส่หรือไม่ก็ได้ ละมันไว้แล้วผลลัพธ์ทั้งหมดจะเป็น partition เดียว การจัดอันดับจึงรันข้ามทุกแถว
- `ORDER BY` ภายใน `OVER` ควบคุม window frame และเป็นอิสระจาก `ORDER BY` สุดท้ายของคิวรี เพิ่มตัวด้านนอกเข้าไปด้วยถ้าคุณสนใจลำดับที่แถวถูกคืนกลับมา
- คุณใช้ window function ใน `WHERE` clause ไม่ได้ เพราะ window ถูกคำนวณหลัง `WHERE` ให้ห่อคิวรีไว้ใน CTE หรือ subquery แล้วกรองบนคอลัมน์ที่คำนวณแล้วจากข้างนอก
- เลือก `ROW_NUMBER` สำหรับลำดับที่เข้มงวด, `RANK` เมื่อค่าเสมอกันควรใช้อันดับร่วมกันและทิ้งช่องว่างไว้, และ `DENSE_RANK` เมื่อค่าเสมอกันใช้อันดับร่วมกันโดยไม่มีช่องว่าง

<Callout title="สวิตช์ทางความคิด">
เมื่อคำขอบอกว่า "สำหรับแต่ละแถว ให้แสดงอันดับ / ยอดสะสม / สัดส่วนของกลุ่มของมันด้วย" วลี "สำหรับแต่ละแถว" นั้นคือสัญญาณของคุณ: คุณต้องใช้ window function ไม่ใช่ `GROUP BY` เก็บแถวไว้ เพิ่มคอลัมน์เข้าไป
</Callout>
